# 바이브코딩 문서 정리 체크리스트

## 새 문서를 만들기 전에
- 기존 canonical source 한 곳에 짧게 추가하는 것으로 끝낼 수 있는가?
- 이번 내용이 다음 세션에도 다시 찾아볼 durable한 사실, 결정, 절차인가?
- `active_task.md`나 `completed/`에 남기면 충분한 일회성 메모는 아닌가?

## 작업 종료 5분 정리
1. 직접 연결되는 canonical source가 갱신됐는지 본다.
2. durable한 메모만 completed, ADR, semantic / procedural 문서로 승격한다.
3. summary layer 문서가 원문 규칙을 다시 설명하고 있으면 포인터로 줄인다.
4. 새 파일을 만들었다면 왜 기존 문서에 못 넣었는지 설명할 수 있어야 한다.
5. 일회성 메모를 남기지 말고 `active_task.md`를 비운다.
