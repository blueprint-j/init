# Retain, Recall, Reflect

## Retain
- 현재 작업의 목표와 blocker는 `docs/plans/active/active_task.md`에 유지한다.
- 안정적인 사실은 의미론적 기억 문서로 보낸다.
- 재참고 가치가 있는 과정과 검증은 일화적 기억으로 남긴다.

## Recall
- 작업 시작 시 `AGENTS.md`를 기준으로 필요한 최소 문서만 읽는다.
- 작업 성격이 분명하면 `docs/agent-rules/`에서 해당 guide를 먼저 찾는다.
- 과거 사례가 필요하면 `docs/plans/completed/`와 `docs/memory/`를 찾는다.

## Reflect
- 작업 종료 시 무엇을 장기 기억으로 승격할지 판단한다.
- 검증된 사실만 의미론적 기억으로 올린다.
- 승격이 끝나면 작업 기억을 비운다.
