# 지식 허브

`docs/knowledge/`는 여러 세션에 걸쳐 다시 참조할 장기 지식만 모아 둔 허브다.

## 핵심 규칙
- 사용자 노출 정책과 문서 운영 기준처럼 다시 찾아볼 가치가 있는 장기 사실만 둔다.
- 내부 wiring과 계층 구조는 `docs/architecture/`에 둔다.
- 현재 작업 상태와 짧은 진행 로그는 `docs/memory/runtime/`에 둔다.
- 새 문서는 기존 문서로 흡수할 수 없을 때만 만든다.

## 이 폴더가 답하는 질문
- 사용자에게 무엇을 약속하고 어떤 정책을 적용하는가?
- 이 저장소의 문서 체계와 장기 기준은 어떻게 유지하는가?

## 이 폴더에 두지 않는 것
- service wiring, repository 조립, 요청 처리 흐름 같은 내부 구조 세부
- 현재 작업 진행 상황, 가정, blocker
- 검증 전의 임시 메모와 조사 로그
