# 지식 허브

`docs/knowledge/`는 여러 세션에서 다시 참고할 장기 기준만 둔다.

## 이 폴더가 다루는 것
- 사용자 노출 기준과 지원 범위
- 문서 체계와 운영 기준
- 프로젝트 성격이 바뀌어도 다시 참고할 사실

## 이 폴더가 다루지 않는 것
- 내부 구조와 서비스 wiring
- 현재 작업의 가정, blocker, 진행 로그
- 검증 전 임시 메모

## 시작 문서
- [product-reference.md](product-reference.md): 사용자에게 드러나는 기준과 장기 정책
- [documentation-reference.md](documentation-reference.md): `docs/` 구조와 운영 기준

## 운영 기준
- 같은 장기 사실을 여러 문서에 중복하지 않는다.
- 기존 문서로 흡수할 수 있으면 새 파일을 만들지 않는다.
