# 프로토콜

AI 에이전트의 행동, 구현, 보안, 검증 원칙을 정의한다.

## 핵심 원칙
* 우선순위는 안전·보안 > 정확성 > 사용자 요청 > 단순함 > 저장소 관례다.
* 목적 달성에 필요한 최소한의 변경만 한다.
* 현재 작업의 shared task spec은 `docs/plans/active/active_task.md` 하나로 유지한다.
* 실행 전에는 관련 코드·설정·문서·테스트를 먼저 확인한다.
* 확인하지 않은 것을 확인했다고 말하지 않는다.
* 불확실성이 크면 질문하고, 그렇지 않으면 가정을 명시하고 진행한다.

## 구현 / 변경
* 요청 범위 안에서만 작업한다.
* 변경으로 인해 필요한 테스트, 문서, 설정 수정은 같은 작업 범위로 본다.
* 부분 수정(diff/patch) 우선, 전체 재작성은 꼭 필요할 때만 한다.
* 자신이 만든 미사용 코드나 죽은 문서는 함께 정리한다.

## 실행 / 디버깅
* 작업을 검증 가능한 목표로 바꾼다.
* 여러 단계가 필요하면 계획과 검증 기준을 먼저 공유한다.
* 가능하면 작게 바꾸고 바로 검증한다.
* 같은 접근으로 3번 연속 실패하면 멈추고 시도 내역과 다음 선택지를 설명한다.
* 반복되는 실패 패턴은 프롬프트 재시도로 덮지 않고 문서 규칙이나 패턴 문서 갱신 대상으로 본다.

## 파일 / 보안 / 의존성
* 신규 파일은 UTF-8을 기본으로 하고 기존 인코딩과 줄바꿈을 유지한다.
* 민감 정보는 코드에 직접 넣지 않는다.
* 새 의존성 추가 전에는 기존 도구로 해결 가능한지 먼저 확인한다.

## 테스트 / 완료 보고
* 퍼블릭 인터페이스와 기대 동작 중심으로 검증한다.
* 실행한 검증은 실제 명령과 핵심 결과로 남긴다.
* `completed/` 기록의 임계값과 형식은 `docs/plans/README.md`를 따른다.
* 실행하지 못한 항목은 `미검증`으로 분리한다.
